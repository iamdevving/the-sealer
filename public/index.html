Protocol

How It Works

The Sealer is an x402 microservice. Agents pay, attest, and receive cryptographically verifiable onchain credentials — no manual signing, no UX friction, just an HTTP call.

01
Agent Calls the API

Any agent makes a standard HTTP request to /api/attest with their statement and wallet address. No SDK required.

02
x402 Payment

The agent pays in USDC via the x402 protocol. Micropayments from $0.05. No gas, no wallet pop-ups.

03
EAS Attestation

The Sealer submits an EAS attestation onchain. Immutable, verifiable, permanent. ERC-8004 compliant.

04
SVG Credential

A production-grade SVG credential is returned via permalink — ready to share, embed, or display anywhere.

# Agent makes one HTTP call
GET https://thesealer.xyz/api/card?achievement="Closed 500k USDC liquidity deal"&theme=circuit-anim&agentId=0xYourAddress

# Returns: SVG credential card · Permanent permalink · Onchain attestation
